# GitHub Project Management Learning Report

## Objective

## 1. GitHub Issues

I learned that GitHub Issues are used to track features, bugs, tasks, and enhancements. Each Issue acts as a work item assigned to a developer. I created multiple feature and bug issues, assigned labels, priorities, and assignees to understand how teams organize work.

### Practical Work

- Created Bug Issues.
- Added labels (feature, bug, priority).
- Assigned Issues to myself.

## 2. Git Branching Workflow

I learned that developers should not work directly on the `main` branch. Instead, a separate branch is created for each feature or bug fix.

### Practical Work
- Created a feature branch.
- Modified project files.
- Committed changes.
- Pushed the branch to GitHub.

## 3. Pull Requests

I understood that a Pull Request is used to request merging code into the main branch after development is completed. It allows code review before merging.

### Practical Work
- Created a Pull Request.
- Added a proper description.
- Linked the Pull Request with an Issue using `Closes #1`.
- Merged the Pull Request.
- Verified that the linked Issue closed automatically.

### Practical Work

I learned that Milestones group related Issues into a sprint or release and help track overall progress.

### Practical Work

## 6. Relationship Between GitHub Features

I understood the complete development workflow used in software teams:

Requirement → Issue → Branch → Development → Commit → Push → Pull Request → Merge → Issue Closed → Project Updated

This workflow helps manage software development efficiently.
